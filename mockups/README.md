# ต้นแบบหน้าจอ Mock-up

ต้นแบบ HTML แบบ static สำหรับสาธิตภาพรวมระบบและแม่แบบหน้าจอตามบทบาท โดยยังไม่เชื่อม Laravel หรือฐานข้อมูลจริง รอบนี้ปรับ UX/UI โดยอิงแนวคิดจาก TPSS ของมหิดล: โครงหน้าผู้ดูแลระบบสีกรมท่า, พื้นที่ทำงาน light mode, ข้อมูลนำหน้า decoration, และสถานะ/คำเตือนที่ชัดเจน

## วิธีเปิด
เปิดไฟล์นี้ใน browser:

```txt
mockups/index.html
```

ถ้าต้องการ icon จาก Lucide ให้เชื่อมอินเทอร์เน็ตตอนเปิดหน้า เพราะ mock-up ใช้ CDN สำหรับ icon เท่านั้น ส่วน UI หลักยังใช้งานได้แม้ icon ไม่โหลด

## สิ่งที่ mock-up จำลอง
- ตัวสลับบทบาท
- fixed navy sidebar พร้อมเมนูตามบทบาทและ badge จำนวนงาน
- ภาพรวมแบบ stats strip 4 ช่อง
- ด่านตรวจก่อนเปิดรอบจัดตาราง
- ข้อมูลกลางแบบหน้าเดียวหลายแท็บ พร้อม filter bar
- ภาระสอน
- หน้าจัดตารางสอนแบบปรับเอง
- ตรวจตารางชนและคุณภาพ
- คำแนะนำการปรับตาราง
- ตรวจสอบโดยกลุ่มสาระ
- รายการรออนุมัติ
- มุมมองตารางหลังเผยแพร่
- รายงาน / พิมพ์ PDF

## บทบาทที่รองรับ
- ผู้ดูแลระบบ
- ฝ่ายวิชาการ
- หัวหน้ากลุ่มสาระ
- หัวหน้างานวิชาการหรือรองฯ วิชาการ
- ครูผู้สอน
- ครูประจำชั้น
- ผู้บริหาร / ผู้ดูแลภาพรวม

## Demo flow ที่แนะนำ
1. เลือกบทบาท `ฝ่ายวิชาการ`
2. เปิด `ภาพรวม` เพื่อดู stats strip และด่านตรวจก่อนเปิดรอบ
3. เปิด `ข้อมูลกลาง` แล้วลองสลับแท็บ `รายวิชา`, `ครู`, `ห้องเรียน`, `ช่วงเวลาเรียน`
4. เปิด `จัดตารางสอน`
5. คลิกช่องตาราง
6. กด `ตรวจตารางชน`
7. กด `วิเคราะห์`
8. เปิด `คำแนะนำ`
9. กลับไป `ส่งอนุมัติ/เผยแพร่` แล้วกด `ส่งอนุมัติ`
10. เปลี่ยนบทบาทเป็น `หัวหน้างานวิชาการ / รองฯ วิชาการ`
11. กด `อนุมัติ`
12. กด `เผยแพร่`
13. เปลี่ยนบทบาทเป็น `ครูผู้สอน`, `ครูประจำชั้น` หรือ `ผู้บริหาร / ผู้ดูแลภาพรวม` เพื่อดูแม่แบบหน้าจอแต่ละบทบาท

## แนวคิด TPSS ที่นำมาใช้
- หน้าภาพรวมไม่ใช่แค่โชว์ตัวเลข แต่เป็นทางลัดไปแก้ข้อมูลที่ขาด
- ข้อมูลกลางเป็นฐานของการสร้างตาราง จึงต้องมีแท็บ ค้นหา ตัวกรอง นำเข้า และสถานะชัดเจน
- ก่อนเปิดรอบจัดตารางต้องมีด่านตรวจข้อมูลสำคัญ
- สีสถานะใช้กับ conflict, warning, success และ info เท่านั้น
- เมนูซ้ายช่วยให้ผู้ใช้ทำงานนาน ๆ โดยไม่หลงตำแหน่ง

## ไฟล์สำหรับเปรียบเทียบ
- `index.html` คือ mock-up ปัจจุบันที่ปรับตามแนว UX/UI จาก TPSS
- `index_before_tpss.html` คือ snapshot ก่อนปรับตามแนว TPSS ใช้เปิดเทียบความแตกต่างของ layout, dashboard และ Master Data
- `index_feedback_v2.html` คือเวอร์ชันหลังแก้ตาม feedback ล่าสุด: เพิ่มเมนูตั้งค่า, ซ่อนแถบขวาในหน้าข้อมูลกลางของผู้ดูแลระบบ/ฝ่ายวิชาการ, ยุบเมนูพิมพ์/PDF ของครู, ปรับตารางเรียน และลดความวุ่นวายของหน้าหลัก

## ขอบเขต
mock-up นี้เน้นภาพรวม UX และตรรกะสิทธิ์การใช้งานเท่านั้น ไม่มี backend, validation จริง, auth จริง หรือ database
