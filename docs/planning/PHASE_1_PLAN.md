# PHASE_1_PLAN.md

## Goal
เตรียมเอกสารและโครงสร้างโปรเจคให้พร้อมสำหรับการนำเสนอหัวข้อ อธิบาย scope ได้ชัด และลดความเสี่ยงก่อนเริ่ม coding phase

## Phase 1 Outputs
- Product vision ชัดเจน
- Requirement แยก functional/non-functional
- Domain rules และ constraints ครบ
- Workflow หลักครบ
- Data model ฉบับ proposal
- Architecture และ module boundary
- Test plan สำหรับ logic สำคัญ
- Backlog แยกเป็น epic/task
- Risk register
- Seed data plan สำหรับ demo
- Proposal outline และคำถามสำหรับอาจารย์

## Recommended Order
1. Finalize PRODUCT.md และ DOMAIN.md
2. Lock MVP scope ใน REQUIREMENTS.md
3. ทำ workflow และ flowchart ให้ตรงกัน
4. ปรับ DATA_MODEL.md ให้รองรับ constraints สำคัญ
5. กำหนด module/service boundary ใน DESIGN.md และ MODULE_MAP.md
6. ทำ constraint test matrix
7. เตรียม seed data และ demo scenarios
8. ทำ proposal outline และ slides

## Phase 1 Exit Criteria
- อธิบายได้ว่าระบบต่างจากโปรแกรมจัดตารางทั่วไปอย่างไร
- อธิบาย hard/soft constraints ได้ชัด
- อธิบายว่าทำไม AI ไม่ใช่ตัวจัดตารางหลักได้
- อธิบายว่าทำไม full auto generator ไม่ใช่ core MVP ได้
- อธิบาย workflow ตรวจสอบ อนุมัติ และเผยแพร่ได้
- มี ERD ระดับ proposal
- มี flowchart ที่สอดคล้องกับ workflow
- มี backlog สำหรับ implementation 4 เดือน
